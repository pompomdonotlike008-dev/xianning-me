app-builder：HTML/CSS/JS → 签名 Android APK

2026-06-27Skill

一句话：把任何单页 HTML/CSS/JS 转成可以直接安装的 Android APK。三重签名，Android 高版本不拒装。

能力

  • 📦 自动生成 Nitron 脚手架（Android WebView 壳工程）
  • 🔨 Gradle 构建，输出 debug + release APK
  • 🔐 uber-apk-signer v1+v2+v3 三重签名
  • 📱 输出文件直接拖到手机上安装

踩过的坑

坑 1：fetch() 对 file:// 无效

APK 内 WebView 用 file:// 协议加载页面，fetch() 请求全部失败。所有页面必须内联为单页 SPA。

坑 2：IndexedDB 不可用

Android WebView 不支持 IndexedDB，需改用 localStorage。涉及数据持久化的逻辑要提前适配。

坑 3：三重签名是必须的

只有 v1 签名 → Android 7+ 警告。只有 v1+v2 → Android 11+ 拒装。必须 v1+v2+v3 全覆盖。

构建流程

1. 接收 HTML/CSS/JS 源文件
2. 生成 Nitron 项目（Android WebView 壳）
3. 将源文件内联注入到 WebView 入口
4. Gradle 构建 release APK
5. uber-apk-signer 三重签名
6. 输出 signed-release.apk

与 hongmeng 的关系

app-builder 是 Android 路径，hongmeng 是鸿蒙路径。两者互补，覆盖双端。同一个 Web App 可以先打出 APK 快速分发测试，验证通过后再走鸿蒙流程上架 AppGallery。
